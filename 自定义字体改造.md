# 自定义字体改造方案

> [PUA生效 🔥] 顶层设计先拉通，底层逻辑要闭环。这份文档是抓手，不是 PPT。

---

## 0. 一句话目标

让用户在 Web 端能 **切换内置字体**（从 `字体/` 目录加载）、**上传自定义字体**、**一键回默认**，全链路本地持久化，零 CDN 依赖（这一项作为附加红利）。

## 2. 顶层设计（Architecture）

### 2.1 数据流闭环

```
┌──────────────┐       ┌───────────────┐       ┌──────────────────┐
│ 用户 UI 选择 │──────▶│ localStorage  │──────▶│ 动态 @font-face  │
│ "选 X 字体"  │       │ 持久化字体ID  │       │ 注入到 <head>    │
└──────────────┘       └───────────────┘       └──────────────────┘
       │                                                 │
       │ 上传自定义                                       ▼
       ▼                                       ┌──────────────────┐
┌──────────────┐       ┌───────────────┐       │ document.body    │
│ POST          │──────▶│ user_data/   │──────▶│ 应用 CSS 变量    │
│ /api/fonts/   │       │ fonts/*.ttf  │       │ --font-sans/mono │
│ upload        │       └───────────────┘       └──────────────────┘
└──────────────┘                │
       │                         ▼
       │                ┌───────────────┐
       └───────────────▶│ GET            │
                        │ /assets/fonts/ │
                        │ <name>         │
                        └───────────────┘
```

### 2.2 字体分类（颗粒度对齐）

| 类别 | 来源 | 可否删除 | 示例 |
|------|------|----------|------|
| **default** | 浏览器默认 / Google Fonts 回退 | ❌ | Fira Sans / Fira Code |
| **builtin** | `字体/` 目录（仓库内置） | ❌ | Minecraft AE / BigNoodleTitling |
| **user** | `user_data/fonts/`（用户上传） | ✅ | 任意 TTF/OTF/WOFF/WOFF2 |

### 2.3 字体角色（slot）

```
--font-sans  → 正文 sans-serif（body 默认）
--font-mono  → 代码/等宽（Fira Code 类）
```

用户可以为每个 slot 独立选字体。例如：正文用 Minecraft AE，代码用 Fira Code。

> 因为信任所以简单 —— 用户的选择就两个槽位，不要做成"每个组件单独选字体"那种过度设计。

## 3. 实施拆解（Sprint Plan）

### Sprint 1: CSS 收敛（半小时 / P0 / 风险 0）

**目标**：把 11 处硬编码 `font-family` 收敛到 `var(--font-sans)` / `var(--font-mono)`。上线后视觉零变化。

**改动清单**：

```css
/* app.css 顶部新增 :root 块 */
:root {
  --font-sans: "Fira Sans", Inter, "Segoe UI", "Microsoft YaHei", Arial, sans-serif;
  --font-mono: "Fira Code", ui-monospace, SFMono-Regular, Consolas, monospace;
}

/* 替换点 */
- body { font-family: "Fira Sans", ...; }
+ body { font-family: var(--font-sans); }

- .code-block { font-family: "Fira Code", ...; }
+ .code-block { font-family: var(--font-mono); }
```

**验收**：跑完后视觉对比 = 0 diff。

---

### Sprint 2: 后端打地基（1.5 小时 / P0 / 风险低）

**新增文件**：`src/mc_mod_i18n/web_fonts.py`

**新增路由**：

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/assets/fonts/<scope>/<name>` | 返回字体二进制（scope=builtin/user） |
| GET | `/api/fonts` | 返回字体清单 JSON |
| POST | `/api/fonts/upload` | multipart 上传 |
| DELETE | `/api/fonts/<name>` | 删除用户字体 |

**`/api/fonts` 响应 schema**：

```json
{
  "default": {
    "sans": { "id": "default-sans", "label": "默认 Fira Sans", "family": "Fira Sans" },
    "mono": { "id": "default-mono", "label": "默认 Fira Code", "family": "Fira Code" }
  },
  "builtin": [
    {
      "id": "minecraft-ae",
      "label": "Minecraft AE",
      "family": "Minecraft AE",
      "url": "/assets/fonts/builtin/Minecraft%20AE.ttf",
      "size_bytes": 16162252,
      "format": "truetype",
      "recommend_slot": "decorative"
    },
    {
      "id": "big-noodle-titling-oblique",
      "label": "Big Noodle Titling Oblique",
      "family": "Big Noodle Titling Oblique",
      "url": "/assets/fonts/builtin/BigNoodleTitling%20Obliqu.ttf",
      "size_bytes": 27989,
      "format": "truetype",
      "recommend_slot": "decorative"
    }
  ],
  "user": [
    { "id": "user-...", "label": "...", "url": "...", "size_bytes": ..., "uploaded_at": "..." }
  ]
}
```

**上传规则**：

| 项 | 约束 |
|----|------|
| 最大体积 | 10 MB |
| 接受扩展名 | `.ttf` / `.otf` / `.woff` / `.woff2` |
| 魔数校验 | TTF: `00 01 00 00` / OTF: `4F 54 54 4F` / WOFF: `77 4F 46 46` / WOFF2: `77 4F 46 32` |
| 文件名 | 服务端 sanitize（去掉 `/`、`\`、`..`、控制字符） |
| 冲突 | 同名追加 `-1`、`-2` 后缀 |

**字体目录**：

```
bundled_resource_root() / "字体" /          # 内置（只读）
user_data_dir() / "fonts" /                  # 用户上传（可写、可删）
```

**核心函数**：

|------|------|
| `fetchFontCatalog()` | GET `/api/fonts` → 渲染 picker |

```json

  "mono": "default-mono"

```

**`@font-face` 注入策略**：动态写入 `<style id="dynamic-fonts">`，每次切换重写整段：

```css
@font-face {
  font-family: "Minecraft AE";
  src: url("/assets/fonts/builtin/Minecraft%20AE.ttf") format("truetype");

}
```

## 4. 关键技术决策（已锁定）

| 决策 | 选项 | 锁定方案 | 理由 |
|------|------|----------|------|
| 用户字体存储 | bundled vs user_data | **user_data_dir()/fonts/** | 跨重装保留，符合 owner 意识 |
| 内置发现 | 扫描 vs 白名单 | **递归扫描 `字体/`** | 灵活，未来加字体直接丢文件即可 |
| 接受格式 | TTF/OTF/WOFF/WOFF2 | **全接受** | 现代浏览器全支持 |
| 拒绝格式 | EOT/SVG | **拒绝** | 已淘汰 |
| @font-face 注入 | 服务端拼 vs 前端动态 | **前端动态 `<style>`** | 省一次往返，热切换零延迟 |
| 大字体处理 | 原样 / subset / 限装饰 | **原样上 + 体积警告 + 异步加载** | 不破坏用户已有资产，UI 上挂提示 |
| Google Fonts | 继续 CDN vs 自托管 | **保留 CDN（本次不动）** | 控制本次 Sprint 边界，避免 scope creep |
| 字体格式探测 | 扩展名 vs 魔数 | **双重校验（魔数为准）** | 安全闭环 |

## 5. 风险登记册

| 风险 | 等级 | 缓解策略 | Owner |
|------|------|----------|-------|
| 16MB 字体首次加载慢 | 🟡 中 | `font-display: swap` + UI 上挂体积警告 | 前端 |
| FOUC 字体加载前样式跳变 | 🟡 中 | `swap` + 启动时 preload 上次选中的字体 | 前端 |
| 用户上传超大/恶意文件 | 🟡 中 | 服务端 10MB 限制 + 魔数校验 | 后端 |
| 字体版权 | 🟡 中 | 上传 UI 挂提示语"请确认你拥有授权" | 前端 |
| 删除当前激活字体 | 🟢 低 | 删除后自动回退默认，localStorage 同步 | 前端 |
| 字体名含特殊字符 | 🟡 中 | sanitize + URL encode | 后端 |
| 浏览器缓存命中过期 | 🟢 低 | 字体响应加 `Cache-Control: public, max-age=31536000, immutable` | 后端 |
| 同名冲突 | 🟢 低 | 服务端自动追加 `-N` 后缀 | 后端 |

## 8. 不在本次范围（Out of Scope）

> 这些是 scope creep 红线，不要拉。

- ❌ Google Fonts 自托管（独立 Sprint，本次不动）
- ❌ Minecraft AE 字体 subset（除非用户明确要求装 fonttools）
- ❌ 字体粗细/字号自定义（只切 family）
- ❌ 字体预览效果（picker 里直接用该字体渲染 label 即可，不做独立预览组件）
- ❌ 字体分享/导出/同步
- ❌ 每组件级别独立字体
