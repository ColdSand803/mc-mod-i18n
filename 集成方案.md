# deep-translator 集成方案

## 背景

当前项目提供 4 种翻译器：`openai-compatible`、`anthropic-compatible`、`glossary`（离线术语表）、`copy`（复制原文）。AI 翻译器依赖 API Key，对零配置场景不够友好。

`deep-translator` 是一个 Python 库（MIT 协议），统一封装了 **13 个翻译引擎**。其中 **3 个免费、无需 API Key、支持句子级翻译**，天然适合零配置场景。

## 核心挑战

Minecraft 官方支持 **127 个语种**（`web.py:4293-4421` `minecraftLocales`），三个引擎各自的代码体系不同：

| 系统 | 代码风格 | 示例 |
|---|---|---|
| Minecraft | `xx_YY` | `zh_cn`, `ja_jp`, `af_za` |
| Google Translate | ISO 639-1 | `zh-CN`, `ja`, `af` |
| MyMemory | ISO 639-1 | `zh-CN`, `ja`, `af`（与 Google 相同） |
| LibreTranslate | ISO 639-1 | `zh`, `ja`, `af` |

> MyMemory 和 Google 共用同一套 ISO 639-1 代码，一套映射两套可用。LibreTranslate 的代码略有差异（如中文用 `zh` 而非 `zh-CN`），需单独处理。

MC 还包含约 **47 个虚构/方言语种**（海盗英语、Klingon、LOLCAT 等），在任何翻译引擎中都无对应，需回退。

## 架构设计

```
web.py (UI) → core.py (任务编排) → translator.py (工厂+接口)
    │                                   │
    │  deep-free 选项                   ├── OpenAICompatibleTranslator (已有)
    │                                   ├── AnthropicCompatibleTranslator (已有)
    │                                   ├── GlossaryTranslator (已有)
    │                                   ├── CopyTranslator (已有)
    │                                   └── DeepTranslator (新增)
    │                                        ├── GoogleTranslator (主力)
    │                                        ├── MyMemoryTranslator (备选)
    │                                        └── LibreTranslator (可选，需自托管)
    │
    └── providerPresets['deep-free']
```

### 多引擎回退链

```
输入文本
  │
  ├─ 1. GoogleTranslator  ──成功→ 返回结果
  │        │失败
  ├─ 2. MyMemoryTranslator ──成功→ 返回结果
  │        │失败
  └─ 3. CopyTranslator (兜底) ──→ 返回原文
```

引擎顺序可通过 `engine_order` 参数配置：`["google", "mymemory"]` / `["mymemory", "google"]` / `["google"]` 等。

## DeepTranslator 接口设计

```python
class DeepTranslator(Translator):
    """包装 deep-translator 多引擎，遵循项目 Translator 接口"""

    def __init__(
        self,
        source_locale: str,         # MC 语种代码，如 "en_us"
        target_locale: str,         # MC 语种代码，如 "zh_cn"
        engine_order: list[str] | None = None,
            # 默认 ["google", "mymemory"]
            # 可选: "google", "mymemory", "libre"
        libre_base_url: str = "",   # Libre 自托管地址（可选）
        proxies: dict | None = None,
        unsupported: str = "copy",  # 'copy' | 'skip' | 'raise'
    ): ...

    def translate_batch(
        self, items: list[TranslationItem]
    ) -> dict[str, str]: ...
        # 按 engine_order 依次尝试，全部失败则复制原文

    def translate_batch_with_failures(
        self, items: list[TranslationItem]
    ) -> tuple[dict[str, str], dict[str, str]]: ...
        # 返回 (成功结果, 所有引擎都失败的项及原因)
```

## 回退策略

| 场景 | 行为 |
|---|---|
| 源/目标语言都支持，Google 正常 | Google 翻译返回 |
| Google 失败（网络/限流） | 自动回退 MyMemory |
| MyMemory 也失败 | 回退 `CopyTranslator`（返回原文） |
| 源语种不支持 | 复制原文，status 标记 `unsupported_source` |
| 目标语种不支持 | 复制原文或抛异常（取决于 `unsupported` 参数） |
| 单条翻译失败 | 标记 `api_failed`，该条返回原文，不阻断其他条目 |

## 不做什么（本期）

- 不包装 PonsTranslator / LingueeTranslator：词级翻译，不适合句子级 MC 文本
- 不包装 ChatGptTranslator / DeepL / Microsoft / 国内引擎：需要 API Key，与现有 AI provider 重叠，留给用户按需组合
- 不新增 UI locale 条目：`deep-free` 复用现有 message key 体系
- 不做并发控制：deep-translator 内部单线程，本项目直接使用

## 测试计划

1. **映射完整性**：127 个 MC locale 都有映射条目；有值的条目在至少一个引擎的支持列表中
2. **Google 翻译**：`en_us→zh_cn`、`ja_jp→ko_kr`、`de_de→fr_fr` 实际调用，验证非空
3. **MyMemory 翻译**：同语种对验证 MyMemory 可用
4. **引擎回退**：模拟 Google 失败 → MyMemory 成功
5. **不支持语种**：`en_us→lol_us` 验证回退为原文复制
6. **Batch 一致性**：输入 N 条返回 N 条，ID 一一对应
7. **现有测试不退化**：`pytest tests/ -v` 全部通过
