# 工具全局动效优化

## 目标

把当前 Web UI 的动效从零散 hover 和 loading 动画，收敛成一套克制、稳定、可复用的工具型动效系统。参考 Behance 优秀案例的精致度，但不采用展示站式的大幅视差、重装饰和长时入场动画。

动效必须服务三个目标：

- 让用户知道当前操作被接收了。
- 让视图切换、详情刷新、状态变化更容易跟上。
- 让上传、预检、翻译、失败、完成等高频流程更有连续感。

## 设计原则

- 快：常规微交互控制在 120-180ms，视图切换控制在 160-220ms。
- 轻：优先使用 `opacity` 和 `transform`，避免影响布局的宽高动画。
- 稳：工具页面不做弹跳、强缩放、长延迟队列和装饰性漂浮元素。
- 可退：完整遵守 `prefers-reduced-motion: reduce`。
- 一致：导航、卡片、列表项、状态提示、详情内容使用同一套节奏变量。

## 动效层级

### 1. 全局节奏变量

新增统一 token：

- `--motion-micro`：极短反馈，例如按钮按下、选项选中。
- `--motion-panel`：面板、视图、详情内容切换。
- `--motion-emphasis`：状态提示、完成/失败等轻强调。
- `--ease-standard`：默认缓动。
- `--ease-emphasis`：轻强调缓动。

### 2. 视图切换

适用范围：

- 主导航：语言文件、报告、硬编码、API 日志。
- 工具级页面：历史、文档、帮助、设置。
- 设置页内分区。

策略：

- 进入时使用轻微 `fade + translateY(6px)`。
- 不做退出动画，避免隐藏逻辑复杂化。
- 主工具页通过统一 `.motion-view-enter` 类触发，而不是复制多份动画。

### 3. 列表与选项

适用范围：

- 文档目录项。
- 问题场景选项。
- 历史任务列表项。
- 设置导航项。

策略：

- hover 使用轻微上浮和边框/背景过渡。
- active 使用短促的背景、边框、阴影过渡。
- 列表项文字必须自然换行，不允许因动效或固定高度溢出。

### 4. 详情内容刷新

适用范围：

- 文档详情正文。
- 帮助快速判断。
- 历史任务详情。
- 结果视图内容。

策略：

- 内容更新后触发一次轻微淡入。
- 滚动区域不改变尺寸，不制造布局跳动。
- 文档详情切换时保持右侧正文独立滚动。

### 5. 状态与加载

适用范围：

- 主状态条。
- provider 测试状态。
- 模型加载状态。
- 任务 loading 卡片。
- 进度条和 lane。

策略：

- 状态提示出现时轻微强调，不用震动。
- loading 保持现有进度流动，但统一节奏。
- 成功/失败状态使用颜色和短淡入，不使用大幅缩放。

## 禁区

- 不做大面积背景动效、粒子、视差、漂浮光斑。
- 不给所有卡片加错峰入场，避免工具界面变慢。
- 不动画化高度、网格列宽、滚动容器尺寸。
- 不让动效遮挡文本、改变信息密度或延迟操作。
- 不牺牲移动端可读性。

## 第一批实施范围

本次先做低风险基础层：

- 增加全局 motion token 和通用 keyframes。
- 给主视图、工具级页面、设置分区添加统一进入动效。
- 给文档详情、帮助预览、历史详情添加内容刷新动效钩子。
- 收敛列表/选项 hover 和 active 的过渡。
- 保留并强化 reduced motion 保护。

后续可继续做：

- 上传和预检阶段的分步骤状态动效。
- 高频错误帮助入口的更明确状态组件。
- 历史任务详情中的 artifact/metric 分组入场。
