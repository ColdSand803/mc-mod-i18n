# 硬编码扫描与工作台

## 什么叫硬编码英文

不是所有英文都来自 `lang/*.json`。

一些文本可能直接写在：

- Java 代码
- Ponder 场景
- 配置注释
- 界面提示

这类内容即使资源包翻译成功，也不会自动变成中文。

## 勾选“扫描硬编码英文”后会得到什么

- `hardcoded-report.html`
  - 用来确认候选文本来源和风险等级。
- `hardcoded-map.template.json`
  - 用来继续补全映射，或导入工作台编辑。

Web UI 完成后还会显示硬编码映射工作台。

## 工作台里能做什么

- 按分类和关键字筛选候选。
- 手工填写 `translation`。
- 导入已有 `hardcoded-map.json` 继续编辑。
- 导出只包含已填写条目的映射文件。

## 为什么导出了映射，但游戏里还是英文

因为 `hardcoded-map.json` 不是资源包语言文件。

它需要被：

- 运行时补丁 Mod
- Mixin
- 配置模板生成器

读取后，才能在正确入口替换文本。

## 什么时候需要先做这一步

- 资源包已经生成，但游戏里仍有零散英文。
- 报告显示资源文件很完整，问题却出现在教程、Tooltip 或配置说明里。
- 你正在处理 Ponder 或整合包配置类内容。
