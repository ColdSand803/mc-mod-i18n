# 翻译记忆与缓存

## 先分清三件事

- `checkpoint / 结果缓存`
  - 作用是避免重复处理同一批输入。
  - 如果你勾选 `忽略缓存并重新翻译`，这一层会被跳过。
- `translation memory`
  - 作用是复用已经确认过的短句或条目翻译。
  - 如果你勾选 `忽略翻译记忆命中`，只会跳过这一层。
- `当前配置 scope`
  - 翻译记忆不是全局无条件复用，而是和当前配置有关。
  - 不同 provider、语言方向、关键输出策略变化，都会影响命中范围。

## 为什么“结果没变化”

常见原因有三类：

- 命中了普通缓存，所以这次任务没有重新请求 provider。
- 命中了翻译记忆，所以旧译文被直接复用。
- 你修改的是不会改变该条结果的参数，例如只是切换了无关设置。

## 什么时候该忽略翻译记忆

- 你刚修过术语表，想确认新术语是否真的生效。
- 你怀疑旧译文质量不佳，但任务总是重复命中旧结果。
- 你从 `deep-free` 切到更高质量 provider，想避免继续复用早期试跑结果。

## 设置页里能做什么

- `刷新记忆`
  - 重新读取当前缓存目录下的翻译记忆统计和预览。
- `导出 JSONL`
  - 导出全部翻译记忆，适合排查或外部整理。
- `压缩去重`
  - 删除重复记录，降低后续维护成本。
- `清理当前配置`
  - 只清理当前 scope 下的记录，风险最小。
- `清理记忆`
  - 清空整个翻译记忆文件，适合大范围重跑前使用。

## 推荐排查顺序

1. 先看结果页或报告里是否提示 `cache hits` / `memory hits`。
2. 如果只是想重测当前 provider，优先勾选 `忽略翻译记忆命中`。
3. 如果想彻底重跑整批内容，再加上 `忽略缓存并重新翻译`。
4. 如果仍然不符合预期，再到设置页检查当前配置下的记忆预览。
