# 结果、报告与任务历史

## 结果页先看什么

- `输出产物`
  - 看是否真的生成了资源包、报告、失败项导出等文件。
- `质量概览`
  - 看成功数、失败数、保留原文、跳过项。
- `性能概览`
  - 看耗时、缓存命中、翻译记忆命中。

## 翻译报告适合做什么

- 搜索失败项和风险项。
- 按状态筛出 `API 失败`、`校验失败`、`不完整`。
- 导出失败项 JSON，便于后续重试或人工处理。

## 任务历史适合做什么

- 找回最近任务的下载链接。
- 打开对应报告继续排查。
- 判断某次任务是不是已经完成、失败或被中断。

如果历史里显示“文件已不存在”，通常说明：

- 任务目录已经被清理或移动。
- 你换了工作目录或缓存目录。
- 当前看到的是旧任务记录，但对应文件已不在本机原位置。

## API 日志什么时候看

以下场景优先打开 API 日志：

- provider 测试连接失败。
- 报告里还有明显缺失项。
- 你怀疑被限流、超时或认证失败。
- 同一个任务反复重试但结果不稳定。

如果结果完全来自缓存，API 日志可能为空，这是正常现象。

## 推荐排查顺序

1. 先看结果摘要，确认是失败、跳过还是命中缓存。
2. 再看翻译报告，定位具体条目。
3. 如果怀疑 provider 问题，再看 API 日志。
4. 需要回溯历史文件时，再到任务历史里确认下载状态。
