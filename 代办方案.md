# mc-mod-i18n 代办方案

## 当前状态

已经完成第一阶段可用版本：

- Python CLI：扫描 Mod JAR 并生成 `zh_cn` 资源包
- 本地 Web UI：上传 JAR、选择翻译器、下载资源包和报告
- Windows exe 打包：PyInstaller one-folder 模式
- JarJar 支持：自动递归扫描 `META-INF/jarjar/*.jar`
- 硬编码英文扫描：输出 `hardcoded-report.html`
- 硬编码映射模板：输出 `hardcoded-map.template.json`

## 近期优先级

### 1. 硬编码翻译工作流

目标：把 Ponder 教程、配置说明、少量 UI literal 从“发现问题”推进到“可翻译”。

待办：

- 在 UI 中增加硬编码报告页筛选：
  - `ponder`
  - `config_comment`
  - `ui_literal`
  - `unknown_literal`
- 支持直接编辑 `hardcoded-map.template.json` 中的 `translation`
- 支持导入已经翻译过的 hardcoded map
- 支持只导出已填写译文的 `hardcoded-map.json`
- 对 hardcoded map 做占位符校验

### 2. Ponder 补丁 Mod 方案

目标：不修改原 Mod，通过额外 patch mod 替换 Ponder literal 文本。

待办：

- 研究 Create / Ponder 在 1.21.1 NeoForge 下的文本显示入口
- 设计 `mc-mod-i18n-ponder-patch` 模板
- 生成补丁 Mod：
  - 内置 `hardcoded-map.json`
  - 通过 Mixin 或事件 Hook 替换 Ponder 文本
- 先支持 NeoForge 1.21.1
- 再评估 Fabric / Forge 版本

风险：

- Ponder API 和类路径可能随 Create 版本变化
- Hook 点需要按版本维护
- 需要真实游戏环境验证

### 3. 配置说明汉化

目标：解决 `config/*.toml` / `serverconfig/*.toml` 中的英文注释。

优先方案：

- 生成汉化配置模板文件夹：

```text
dist/config-cn/
  aeronautics-server.toml
  offroad-server.toml
  simulated-server.toml
```

待办：

- 从 hardcoded `config_comment` 生成翻译映射
- 支持扫描已有 config 文件并替换注释
- 输出替换后的配置模板
- 保留原配置值，只替换注释文本

后续方案：

- 做配置注释运行时补丁，但默认不启用

### 4. 翻译质量

目标：从“能生成”提升到“可用汉化包”。

待办：

- 增加 OpenAI 兼容批量翻译的失败重试
- 翻译缓存：
  - 按 `source + key + category` 缓存
  - 避免重复花费
- 术语表增强：
  - Create 术语
  - 机械动力常用词
  - 单位和缩写白名单
- 二次校验：
  - `%s`
  - `%1$s`
  - `{0}`
  - `§a`
  - 换行
  - 数字单位

## 不建议近期做

- 直接修改原 Mod JAR
- 直接修改 `.class` 字节码
- 自动替换所有硬编码字符串
- 无人工确认地处理 `unknown_literal`

这些方案风险高，容易破坏 Mod 行为。当前路线应保持：

```text
标准 lang -> 资源包
Ponder 硬编码 -> 补丁 Mod
配置注释 -> 汉化配置模板
```
